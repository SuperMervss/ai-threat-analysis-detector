Week 10 — Error Handling

Implemented validation to catch missing or empty input fields (for example, `input_text` or `threat_description`) before calling the Flowise scoring model. A validation `Code` node inspects required fields and sets a human-readable `error_reason`; the `IF` node routes invalid items down the error branch where an Airtable Update sets `Alerts.status` to `error`, writes `error_reason`, and preserves the original `record_id`. Valid items continue to the HTTP Request node; the `Merge` node is configured to `combineByPosition` so responses align positionally with the original records, allowing the `Normalizing Output` node to create `Scoring_Results` and update the original alert to `scored`. This prevents unnecessary LLM calls, provides auditable error logging for triage, and preserves writeback linkage for troubleshooting.
