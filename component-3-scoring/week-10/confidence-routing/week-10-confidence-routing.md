We selected an 80% confidence threshold because it balances automation and safety: empirically it retained high‑quality LLM outputs (reducing human review load) while still flagging borderline results for manual inspection, it maps naturally to our normalized 0–100 confidence scale and is easy for analysts to understand, and it provides a conservative cutoff that minimizes false positives from the model. The threshold is configurable — monitor initial runs for precision/recall tradeoffs and adjust (or add an intermediate “escalate” bucket) if operational data shows the model is over‑ or under‑confident.
